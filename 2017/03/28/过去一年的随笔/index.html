过去一年的随笔

上一年就想写博客了，记录下自己走过的路，虽然走了比较多的弯路，直到现在才开始写，也是严重的拖延症。

作为一个进入职场不久的弱鸡，以我自身的经历来给大家作参考，希望对一些像我这样的大兄弟或大妹子有帮助。在这里，给各位正在找工作的同学一些个人建议。面试时可以将自身能力强行装到1.5倍，然后在试用期将自身的能力提升到面试时的期望值。

相信大多数的人都是在大学的时候贼颓废，各种宅，各种撸机，各种煲剧，各种无聊。等到大四要找工作时才意识到社会的现实，为找工作日渐消瘦。我也浪费了大学前三年的时间，宅在宿舍打游戏，从大三下学期开始先学习iOS开发，那时候iOS开发被吹得都要上天了，然后自己基本没学到什么东西，就出去找工作了。作为一个广东省中等院校本科生（本人比较颓废就不给母校丢脸了），也是历经重重困难才在秋招快结束的时候先找到两份实习的工作机会，然后因为薪资的问题就没有去实习。在这里，我奉劝各位正在找工作的大兄弟，如果一个公司愿意给你实习的机会，你真的不要在意其他的，互联网这个行业，没有工作经历和有过实习经历是差距很大，如果你是在大学就已经很牛逼了，有比较牛逼的项目作品，就请随意。你要将实习时的项目理解透彻，如果你要跳槽换工作，这个对你很有帮助，不要只接触自己负责的模块。

我的第一份工作是在深圳，感受到了深圳飞快的生活节奏。我很感谢当初在深圳实习的公司，给予很多时间让我们自己去学习，上班很闲，可以自由去学习技术。让我得到很大锻炼的是，让我一个新手带着另外一个小白去开发一个项目。虽然项目相对比较简单，但是熟悉了项目从零开始开发到发布的开发流程，相当于具备了独立开发的能力。

第二份工作是在广州，刚来上班感觉压力挺大的，毕竟面试时展示的1.5倍的实力。刚开始两个月上班很忙，晚上也在进修。等到自己熟悉当前薪资的技能水平，就慢慢发觉自己的工作量少了，其实这是自身能力提升了，效率加快了，造成了错觉。

作为一个有点追求的人，当发现自己闲下来了，发现自身在当前公司已经没有发展空间了，就可以考虑跳槽了。跳到更高的地方，对技能有着更高的要求，然后让自己融入高层次。这种是比较累的做法，找工作累，刚去上班更累，提升自身融入层次最累。工作可以闲，但是自己不能给自己放假，要把握每一天时间来学习。

我一直都是在写业务层的代码，自我感觉在业务和逻辑层已经有一定的造诣。现在开始研究底层技术，现阶段先研究runtime和RunLoop，音频，本人也是处在进阶的阶段，希望大神可以赐教。

终于开始经营自己的博客了，从下一篇文章开始写自己在工作中的技术心得和经验，尽量争取每月可以产出两篇并且具备一定质量的文章。很累，但是要坚持。
